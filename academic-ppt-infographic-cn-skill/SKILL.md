# 中文学术科技奖励PPT信息图生成技能

## 1. 适用场景

当用户提出以下需求时使用本技能：

- 根据技术文字生成学术风格PPT图；
- 参考已有PPT页面，替换或更新其中某一局部图示；
- 将复杂技术拆解为两栏、三栏或四层结构；
- 生成科技成果鉴定、科技奖申报、项目答辩风格的技术路线图；
- 保持原页面标题、顶部说明、底部结论不动，只修改中间图；
- 生成16:9横版PNG，并可进一步制作成可编辑PPT。

本技能优先适用于电力电子、储能、电网控制、变流器、构网型技术等工程技术主题，也可迁移到其他理工科领域。

---

## 2. 核心风格定位

目标风格：**正式、学术、工程化、逻辑清晰、适合专家评审快速阅读**。

视觉特征：

- 白色或极浅灰背景；
- 深蓝色主标题与栏目标题；
- 红色仅用于“风险、失配、不一致、下降、困难”等问题强调；
- 蓝色用于“结构、流程、控制、结论”；
- 绿色可用于“技术措施、效果、优化结果”；
- 采用细灰虚线或浅蓝分隔线，不使用粗重外边框；
- 以流程图、控制框图、等效电路、设备图标、波形图和指标卡片为主要视觉元素；
- 一页只表达一条主逻辑，避免装饰性元素喧宾夺主。

## 3. 标准工作流

### 步骤1：锁定不可修改区域

对于“只改中间图”的任务，先明确三个区域：

1. **固定区**：标题、顶部说明文字、底部结论条、页码；
2. **重绘区**：用户指定的中间图示区域；
3. **风格继承区**：原页面字体、颜色、边距、线条粗细、栏目标题样式。

提示词中必须明确：

> 保持顶部标题区、顶部说明文字、底部深蓝结论条和页码不变，仅重绘中间区域。

### 步骤2：将技术内容压缩成视觉逻辑

先把长文本提炼为：

- 每个栏目1个核心难点；
- 每个栏目2—4个机制或原因；
- 每个栏目1句底部总结；
- 红色词不超过该栏文字的20%；
- 单个标签尽量控制在4—10个汉字。

典型结构：

```text
总挑战
├─ A：同步难
│  ├─ 时间同步
│  ├─ 相位同步
│  └─ 载波同步
├─ B：均流难
│  ├─ 线路阻抗差异
│  ├─ 滤波参数失配
│  └─ 环流与电流不均衡
└─ C：功率分配难
   ├─ 站控指令层
   ├─ 协调决策层
   ├─ 参数计算层
   └─ 下发执行层
```

### 步骤3：选择页面构图

优先使用以下构图：

#### 构图A：三栏挑战图

适合三个并列难题：

- 左：同步；
- 中：均流；
- 右：功率分配；
- 栏间使用细灰虚线分隔；
- 每栏顶部深蓝圆角标题条；
- 每栏底部放一条“原因 → 后果”的总结框。

#### 构图B：双层技术图

适合“硬件补偿 + 软件调节”：

- 左侧：硬件拓扑、铜排、滤波、采样；
- 右侧：下垂控制、虚拟阻抗、均流PI；
- 中间用箭头表示“参数失配 → 动态均流”；
- 底部放工程价值。

#### 构图C：挑战—技术—效果

适合创新点总览：

- 左：痛点；
- 中：核心技术；
- 右：指标与工程效果；
- 强调“输入问题—技术闭环—输出指标”。

### 步骤4：组织AI生成提示词

提示词必须包含：

1. **编辑对象**：明确以参考图片为编辑目标；
2. **固定内容**：哪些区域不能改；
3. **修改范围**：只改哪里；
4. **布局结构**：几栏、每栏标题；
5. **图形元素**：设备、控制框图、波形、等效电路；
6. **文字清单**：需要出现的标签；
7. **颜色逻辑**：蓝色主结构、红色问题、绿色技术；
8. **禁止项**：不增加粗红框、不改页脚、不生成多余文字；
9. **画幅**：16:9横版；
10. **质量要求**：学术、简洁、可读、无标签错误。

完整模板见：`templates/image_edit_prompt.md`。

### 步骤5：首轮生成后检查

重点检查：

- 标题是否被改动；
- 页码是否保留；
- 底部结论是否被改写；
- 中文标签是否错字、重字或缺字；
- PCS编号是否顺序正确；
- 公式上下标是否清晰；
- 是否出现不需要的红色外框；
- 栏目之间是否视觉均衡；
- 红色是否过多；
- 图是否像PPT，而不是海报或插画。

### 步骤6：局部修订

修订时不要重新描述整页，只明确：

- “保留全部布局”；
- “只修改某一标签”；
- “取消某条边框”；
- “纠正PCS1—PCS4编号”；
- “不改变顶部和底部”。

示例：

> 保持整页布局、文字、颜色与比例不变，仅删除中间区域外围的红色矩形边框，其他元素不得移动或改变。

### 步骤7：交付

默认交付：

- 16:9 PNG；
- 必要时附可编辑PPTX；
- 对于AI生成图，建议最终在PPT中覆盖关键中文文字和公式，以确保绝对准确与可编辑性。

## 4. 中文文字层级规范

| 层级 | 建议字号（PPT） | 样式 |
|---|---:|---|
| 页面标题 | 26—32 pt | 深蓝、加粗 |
| 顶部说明 | 17—22 pt | 深蓝/红色关键词 |
| 栏目标题 | 16—20 pt | 白字、深蓝底 |
| 模块标题 | 13—17 pt | 深蓝/绿色、加粗 |
| 标签 | 11—15 pt | 深灰或蓝色 |
| 底部结论 | 20—26 pt | 黄色或白色、深蓝底 |

### 文字精炼规则

- “多机并联精准同步控制技术”可缩为“多机精准同步”；
- “分布式并联自主对等均流控制技术”可缩为“自主对等均流”；
- “分层协调功率分配控制技术”可缩为“分层协调功率分配”；
- 技术机制优先使用名词短语，不写长句；
- 每栏底部用“原因 → 后果”形成评审记忆点。

## 5. 工程技术图形语言

### 同步类

使用：

- 时钟图标；
- 错位方波；
- 相位错开的正弦波；
- 错位三角载波；
- 多PCS接入公共母线。

推荐标签：

- 时间同步；
- 相位同步；
- 载波同步；
- 同步精度不足；
- 同频同相难；
- 高频差模环流风险。

### 均流类

使用：

- 多PCS并联等效电路；
- 不同线路阻抗`Z1、Z2、Zn`；
- 不同滤波参数`L1/C1、L2/C2`；
- 红色虚线环流箭头；
- 两组不同幅值的电流波形。

推荐标签：

- 线路阻抗差异；
- 滤波参数失配；
- 环流；
- 电流不均衡；
- 电流均衡度下降；
- 环流损耗增加。

### 分层功率分配类

使用：

- 四级竖向流程；
- 从执行层分叉至多个PCS/电池簇；
- SOC、SOH、温度、容量、响应速度图标；
- 功率响应不一致提示。

推荐标签：

- 站控指令层；
- 协调决策层；
- 参数计算层；
- 下发执行层；
- 多目标、多约束；
- 快速、公平、稳定功率分配。

## 6. 图像生成与PPT制作的分工

### 适合直接用AI生成

- 复杂技术示意图初稿；
- 三栏信息图；
- 设备+波形+控制结构的组合图；
- 原PPT局部重绘。

### 适合在PPT中后期处理

- 关键指标数字；
- 公式；
- 专利号、论文名、标准号；
- 页码；
- 需要绝对准确的中文标签；
- 需要后续修改的文本框与箭头。

推荐流程：

```text
AI生成视觉底图
→ 在PPT中裁剪/排版
→ 覆盖关键文字与公式
→ 对齐、统一字体
→ 导出PNG进行最终检查
```

## 7. 常见问题与修复

### 问题1：AI生成中文错字

处理：

- 将长句拆为短标签；
- 只保留关键字在图中；
- 后期在PPT中覆盖文字；
- 对单个错字进行局部编辑，而不是重做整页。

### 问题2：页面像宣传海报，不像学术PPT

处理：

- 降低渐变和阴影；
- 使用细线、浅底色、矩形框；
- 增加控制框图、公式、波形和电路；
- 减少大图标与装饰插画。

### 问题3：信息过密

处理：

- 每栏只保留3个要点；
- 底部结论只保留1句；
- 技术文字移到讲稿，不全部塞进图中；
- 将“解释”改为“标签+箭头”。

### 问题4：红色过多

处理：

- 红色只表示问题、失配、风险与下降；
- 正常流程全部使用蓝色；
- 技术改进使用绿色；
- 红色面积控制在页面总面积的10%以内。

### 问题5：多余红色外围框

处理：

在编辑提示词中明确：

> 删除中间区域外围的红色矩形框线，保留内部红色关键词和环流箭头，其他元素不变。

## 8. 输出质量底线

最终页面应满足：

- 专家在5秒内看懂三类难点；
- 页面存在清晰的左—中—右或上—下逻辑；
- 关键结论可脱离讲稿独立理解；
- 所有PCS编号、变量、单位、百分比准确；
- 不出现中文乱码、错别字、重复编号；
- 顶部与底部原始内容未被误改；
- 图片分辨率不低于1600×900；
- 插入PPT后在100%缩放下仍清晰。

## 9. 可复用的一句话指令

### 生成新图

> 按中文科技奖励答辩PPT风格，将以下技术内容生成16:9学术信息图：白底、深蓝主标题、红色突出问题、蓝色表示流程、绿色表示技术措施；采用三栏结构，每栏包含标题、技术示意图和“原因→后果”总结，整体简洁、工程化、适合专家评审。

### 局部替换

> 以参考PPT截图为编辑目标，保持顶部标题、说明文字、底部深蓝结论条和页码完全不变，仅重绘中间图示区域；继承原页面字体、颜色、边距和栏目标题样式，不增加外框，不移动固定内容。
